set和multiset的区别

std::set 和 std::multiset 是 C++ 标准库中的两个关联容器，它们都用于存储一组有序且唯一的元素。它们之间的主要区别在于元素的唯一性和插入操作的特点。

std::set：

  • std::set 是一个集合容器，其中的元素是唯一的，不允许重复值。
  • 在 std::set 中插入元素时，如果已经存在相同的元素，则插入操作会被忽略，不会导致重复元素的出现。
  • std::set 中的元素会根据元素的值自动进行排序，默认使用元素类型的 < 操作符来进行比较。
  • 查找和插入的平均时间复杂度为 O(log n)，其中 n 是 std::set 中元素的数量。

std::multiset：

  • std::multiset 是一个多重集合容器，其中的元素允许重复值，允许插入多个相同的元素。
  • 在 std::multiset 中插入元素时，插入操作总是会成功，不会忽略重复元素。
  • std::multiset 中的元素会根据元素的值自动进行排序，默认使用元素类型的 < 操作符来进行比较。
  • 查找和插入的平均时间复杂度为 O(log n)，其中 n 是 std::multiset 中元素的数量。

插入元素：

  1. **insert()**： 这个成员函数用于将元素 value 插入到 std::set 容器中。如果容器中已经存在相同的元素，则插入操作会被忽略，因为 std::set 中的元素是唯一的。insert() 返回一个 std::pair 对象，其中的 first 成员是指向插入元素（或已经存在的相同元素）的迭代器，second 成员是一个 bool 值，表示插入是否成功（true 表示插入成功，false 表示插入失败）。

删除元素：

  1. **erase()**：

    这个成员函数用于从 std::set 容器中删除元素。它有多个重载版本，可以根据迭代器或元素值进行删除操作。

    • erase(iterator position)：删除指定迭代器 position 所指向的元素，并返回指向被删除元素后面元素的迭代器。
    • erase(const key_type& value)：删除容器中值为 value 的元素，并返回删除的元素数量（0或1，因为 std::set 中元素是唯一的）。

unordered_set

std::unordered_set 是 C++ 中标准库提供的哈希集合（无序集合）容器，用于存储唯一的元素，且元素是无序的。它是基于哈希表（散列表）实现的，能够提供快速的插入、删除和查找操作，平均时间复杂度为 O(1)。

哈希集合（unordered set）与有序集合（set）不同，它不会维护元素的插入顺序，因此元素的顺序是无序的。这使得 std::unordered_set 适用于需要高效查找且不需要元素有序的场景。

std::unordered_set 的特点：

  • 快速查找：通过哈希函数将元素映射到内部的存储桶（bucket），可以在平均情况下以常数时间复杂度 O(1) 查找元素。
  • 唯一性：std::unordered_set 中不允许存在重复的元素，每个元素只能出现一次。
  • 插入和删除效率高：插入和删除元素的时间复杂度也为常数时间复杂度 O(1)。
