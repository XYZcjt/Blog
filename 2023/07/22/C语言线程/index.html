线程和进程

进程（Process）和线程（Thread）都是计算机中实现并发执行的机制，但它们有一些重要的区别和对比：

  1. 定义：

    • 进程：进程是计算机中正在运行的程序的实例。每个进程都有独立的内存空间、程序计数器、寄存器等资源，相互之间是隔离的。进程是系统进行资源分配和调度的基本单位。
    • 线程：线程是进程中的执行单元。一个进程可以包含多个线程，这些线程共享进程的地址空间和资源，但有自己的栈空间和线程计数器。

  2. 资源开销：

    • 进程：由于进程之间的隔离性，切换进程的开销较大。每个进程有独立的内存空间，创建、销毁进程需要较多的资源开销。
    • 线程：由于线程共享进程的地址空间和资源，切换线程的开销较小。创建和销毁线程的开销较小。

  3. 通信：

    • 进程：进程之间通信需要通过进程间通信（Inter-Process Communication，IPC）机制，如管道、共享内存、消息队列等。
    • 线程：线程之间通信更加方便，可以直接读写共享的全局变量或使用互斥锁等同步机制。

  4. 并发性：

    • 进程：进程之间并发执行，互不干扰。
    • 线程：线程在同一个进程中并发执行，共享进程的资源，需要注意同步和互斥问题。

  5. 稳定性：

    • 进程：由于进程之间的隔离性，一个进程的崩溃不会影响其他进程。
    • 线程：一个线程的崩溃可能导致整个进程崩溃。

  6. 调试：

    • 进程：由于进程之间隔离，调试相对容易。
    • 线程：线程共享进程的资源，调试相对复杂，可能涉及多个线程的交互。

总体而言，进程适合在需要隔离和稳定性较高的情况下使用，但资源开销较大。线程适合在需要共享资源和较小的资源开销下使用，但需要注意同步和并发控制问题。

线程的创建和定义

在 C 语言中，使用 POSIX 线程库（pthread）来创建和管理线程。以下是线程的创建和定义步骤：

  1. 包含头文件：首先，需要包含 pthread.h 头文件，它包含了 POSIX 线程库的相关函数和数据类型。
  1. 定义线程函数：在创建线程之前，需要先定义线程的执行函数，该函数会在新创建的线程中执行。线程函数的原型是 void *func(void *arg)，它接受一个 void* 类型的参数，并返回一个 void* 类型的结果。
  1. 创建线程：使用 pthread_create 函数创建一个新的线程。该函数接受四个参数：线程标识符指针、线程属性、线程函数指针和传递给线程函数的参数。
  1. 等待线程结束：如果需要等待新创建的线程结束，可以使用 pthread_join 函数。这将阻塞当前线程，直到指定的线程终止。

Race condition(竞态条件)和锁的应用

在 C 语言中，使用锁（互斥锁）是解决并发问题的一种常见方法，它可以保证在同一时间只有一个线程可以访问共享资源，从而避免了竞态条件和数据一致性问题。下面是使用锁解决并发问题的基本步骤：

  1. 包含头文件：首先，需要包含头文件 pthread.h，它包含了 POSIX 线程库的相关函数和数据类型，包括锁的操作函数。
  1. 初始化锁：在使用锁之前，需要创建并初始化一个互斥锁。
  1. 加锁和解锁：在访问共享资源之前，需要先加锁；在完成共享资源的访问后，需要解锁。
  1. 销毁锁：在程序结束前，需要销毁互斥锁。

使用锁的典型场景是多个线程同时修改一个共享数据结构或变量。在这种情况下，使用互斥锁可以确保在同一时间只有一个线程可以修改该数据，避免了数据不一致或损坏。

False sharing(伪共享)

False sharing（伪共享）是一种并发编程中常见的性能问题，它发生在多个线程同时访问共享的内存区域，但实际上每个线程只修改了自己所需的部分数据，却导致其他线程缓存中的数据无效，从而降低了程序的性能。

False sharing 的典型场景是多个线程同时访问位于同一个缓存行（cache line）的不同数据，由于缓存一致性协议的作用，一个线程修改了其中一个数据，会导致整个缓存行的数据失效，其他线程必须重新从内存中加载整个缓存行，即使其他线程没有修改过这些数据。

pthread_exit和pthread_join的联动

pthread_exit() 和 pthread_join() 是 POSIX 线程库中用于管理线程的两个重要函数，它们可以结合使用来实现线程的终止和等待。下面是它们的联动关系：

  1. pthread_exit() 函数：
    • pthread_exit() 用于在线程中显式地终止当前线程的执行。
    • 当线程调用 pthread_exit() 时，它会立即退出，但不会影响其他线程的执行。
    • pthread_exit() 可以传递一个指向线程返回值的指针作为参数，它会将这个返回值传递给调用它的线程。
  2. pthread_join() 函数：
    • pthread_join() 用于等待指定线程的结束，主线程可以调用它等待子线程的完成。
    • 当主线程调用 pthread_join(thread_id, &status) 时，它会阻塞等待 thread_id 线程的终止，并且可以获取到pthread_exit() 返回来的值，将其存储在 status 变量中。
    • 如果不关心子线程的返回值，可以将 status 设置为 NULL。

线程分离

死锁

线程死锁是指两个或多个线程在争夺资源时发生了相互等待，导致所有线程都无法继续执行的情况。在多线程编程中，死锁是一种常见的问题，它会导致程序无法正常运行，造成系统资源的浪费。

为了避免线程死锁，需要合理地管理锁的获取顺序，并确保所有线程在获得锁时都能快速释放锁。另外，可以使用互斥锁、条件变量等同步机制来协调线程的执行，以避免死锁的发生。

读写锁

在 C 语言中，读写锁（Read-Write Lock）是一种用于控制对共享资源进行读写访问的同步机制。读写锁允许多个线程同时读取共享资源，但只允许一个线程进行写操作，以提高并发性能和避免写操作之间的竞争条件。

读写锁有两种状态：读取状态和写入状态。多个线程可以同时获取读取锁（读取状态），以允许并发读取操作。当一个线程获取写入锁（写入状态）时，其他线程无法获取读取锁或写入锁，以确保资源在写入期间不会被其他线程读取或写入。

在 C 语言中，读写锁的相关函数可以在 pthread.h 头文件中找到。常用的读写锁函数包括：

  1. pthread_rwlock_init(): 初始化读写锁。
  2. pthread_rwlock_destroy(): 销毁读写锁。
  3. pthread_rwlock_rdlock(): 获取读取锁（共享锁），允许多个线程同时读取资源。
  4. pthread_rwlock_wrlock(): 获取写入锁（独占锁），只允许一个线程进行写入操作。
  5. pthread_rwlock_unlock(): 释放读写锁。
  6. pthread_rwlock_tryrdlock(): 尝试获取读取锁，如果锁已经被写入锁占用，则立即返回失败。
  7. pthread_rwlock_trywrlock(): 尝试获取写入锁，如果锁已经被读取锁或写入锁占用，则立即返回失败。

条件变量

在 C 语言中，线程条件变量的函数可以在 pthread.h 头文件中找到，并在使用 POSIX 线程库时进行调用。这些函数用于实现线程间的同步和通信，特别是与互斥锁（Mutex）一起使用以确保线程安全。

常用的线程条件变量函数包括：

  1. pthread_cond_init(): 初始化条件变量。用于将条件变量与一个特定的属性对象关联起来。

  2. pthread_cond_destroy(): 销毁条件变量。用于销毁已经初始化的条件变量。

  3. pthread_cond_wait(): 等待条件变量满足。使线程等待并释放互斥锁，直到接收到信号或广播通知。

  4. pthread_cond_signal(): 发送信号。用于向一个等待条件变量的线程发送信号，唤醒其中一个线程。

  5. pthread_cond_broadcast(): 广播通知。用于向所有等待条件变量的线程发送信号，唤醒所有线程。

  6. pthread_cond_timedwait(): 定时等待。类似于 pthread_cond_wait()，第三个参数表示线程阻塞的时长，但是需要额外注意一点：struct timespec这个结构体中记录的时间是从1971.1.1到某个时间点的时间，总长度使用秒/纳秒表示。

信号量

在 C 语言中，信号量是一种用于线程间同步和互斥的计数器。它是 POSIX 线程库提供的一组函数，用于实现进程或线程之间的同步和互斥。信号量主要用于控制对共享资源的访问，以避免竞争条件和并发问题，使用信号量要添加头函数#include <semaphore.h>。

信号量函数的主要操作包括：

  1. sem_init(): 初始化信号量。用于创建并初始化信号量。

    参数说明：

    • sem：指向要初始化的信号量的指针。
    • pshared：用于指定信号量的共享方式。
      • 如果 pshared 参数为 0，表示线程共享。
      • 如果 pshared 参数为非零值，表示进程共享。
    • value：用于初始化信号量的值。信号量的值必须是一个非负整数。

    返回值：

    • 如果成功，返回 0。
    • 如果出现错误，返回 -1，并设置 errno 来指示错误类型。

  2. sem_destroy(): 销毁信号量。用于销毁已经初始化的信号量。

  3. sem_wait(): 等待信号量。当信号量的值大于 0 时，将信号量的值减 1；如果信号量的值为 0，则阻塞线程等待。

  4. sem_post(): 发送信号。将信号量的值加 1，并唤醒等待该信号量的线程。

  5. sem_trywait(): 尝试等待信号量。类似于 sem_wait()，但是不会阻塞线程，如果信号量的值为 0，则立即返回错误。

  6. sem_timedwait(): 定时等待信号量。类似于 sem_wait()，但是增加了一个超时时间，如果在超时前未获取到信号量，则返回错误。
