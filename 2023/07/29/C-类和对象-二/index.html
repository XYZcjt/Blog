综上所述，C++中继承的基本语法是使用派生类声明中的 public、protected 或 private 关键字指定继承类型，通过继承，派生类可以拥有基类的所有属性和行为，并可以通过对象调用基类的成员函数。

继承的方式

在C++中，继承有三种方式：公有继承、保护继承和私有继承。这些继承方式决定了派生类从基类继承成员的访问权限。下面分别介绍这三种继承方式，并提供相应的例子：

  1. 公有继承（Public Inheritance）：

    在公有继承中，派生类从基类继承的成员在派生类中的访问权限与基类中的访问权限相同。基类中的公有成员在派生类中仍然是公有成员，基类中的保护成员在派生类中仍然是保护成员，但基类中的私有成员在派生类中是不可访问的。

  2. 保护继承（Protected Inheritance）：

    在保护继承中，基类的公有和保护成员在派生类中都变成了保护成员，私有成员仍然不可访问。

  3. 私有继承（Private Inheritance）：

    在私有继承中，基类的公有和保护成员在派生类中都变成了私有成员，私有成员仍然不可访问。

总结：

  • public 继承：基类的公有成员和保护成员会继承为派生类的公有成员和保护成员，基类的私有成员不可直接访问。
  • protected 继承：基类的公有成员和保护成员会继承为派生类的保护成员，基类的私有成员不可直接访问。
  • private 继承：基类的公有成员和保护成员会继承为派生类的私有成员，基类的私有成员不可直接访问。

注意：

  • 父类中的私有成员也被继承下去，只是由编译器隐藏后看不到了。

继承中同名成员的处理

在 C++ 继承中，如果派生类和基类中出现了同名的成员变量或成员函数，编译器会根据访问权限和继承类型来处理这些同名成员。

  1. 同名成员变量的处理：

    • 如果派生类中定义了和基类相同名称的成员变量，则派生类的成员变量会覆盖基类的同名成员变量。这就是所谓的隐藏（隐藏基类的同名成员变量）。在派生类中访问该成员时，会访问派生类自己的成员变量。
    • 如果派生类想要访问基类的同名成员变量，可以通过作用域解析运算符 :: 来访问，例如 Base::member。

  2. 同名成员函数的处理：

    • 如果派生类中定义了和基类相同名称的成员函数，该成员函数会覆盖基类的同名成员函数。这就是所谓的函数重写（Function Overriding）或覆盖。在派生类中调用该函数时，会调用派生类自己的成员函数，而不是基类的同名成员函数。
    • 如果希望在派生类中调用基类的同名成员函数，可以使用作用域解析运算符 :: 来指定调用基类的函数，例如 Base::function()。

菱形继承

菱形继承（Diamond Inheritance）是一种特殊的多继承情况，在这种情况下，一个派生类通过多条路径继承同一个基类，导致同一个基类的成员在派生类中出现多次。这可能会引发一些问题，其中最常见的问题是二义性（Ambiguity），即编译器无法确定访问哪个同名成员。

现在问题来了，Child 类继承了两个 Parent 类，而每个 Parent 类都继承自 Grandparent 类。因此，Child 类实际上同时继承了两份 Grandparent 类的成员。当我们在 main() 函数中调用 child.display() 时，编译器无法确定应该调用哪个 display() 函数，因为它们在 Child 类中都存在。

为了解决这个问题，C++ 提供了虚继承（virtual inheritance）的机制。通过使用 virtual 关键字声明虚继承，可以确保同一个基类在继承链中只出现一次，从而解决了菱形继承问题。虚继承在继承链中创建一个共享的基类子对象，所有继承自该基类的派生类共享这个基类子对象，避免了重复继承造成的二义性。

多态

C++ 支持两种多态：静态多态（静态多态性，也称为编译时多态性）和动态多态（动态多态性，也称为运行时多态性）。

多态性是通过虚函数实现的。多态性发生在使用基类指针或引用来调用虚函数的时候。具体来说，多态性在以下情况下发生：

  1. 当一个类包含虚函数时，如果派生类对该虚函数进行了重写（覆盖），并且通过基类指针或引用调用虚函数时，会根据指针或引用所指向的对象的实际类型来动态决定调用哪个派生类的虚函数。
  2. 当使用基类指针或引用指向派生类对象，并通过这些指针或引用调用虚函数时，会发生多态性。此时，程序会根据对象的实际类型在运行时动态地确定调用的虚函数，从而实现了多态性。

静态多态

静态多态是通过函数重载和运算符重载实现的。在编译时，编译器根据函数或运算符的参数类型来确定调用具体的函数或运算符实现。编译器在编译阶段就确定了调用哪个函数或运算符，因此称为静态多态。静态多态在运行时没有额外的开销，因为函数或运算符的调用在编译时就已经决定。

静态多态和动态多态的区别

实现机制：

  • 动态多态：通过虚函数实现。在基类中声明虚函数，在派生类中进行重写（覆盖）。通过基类指针或引用调用虚函数时，实际调用的是基类指针或引用所指向的对象的类型的函数，而不是基类的函数。动态多态在运行时根据对象的实际类型动态决定调用的函数。
  • 静态多态：通过函数重载和运算符重载实现。在编译时根据函数或运算符的参数类型来确定调用具体的函数或运算符实现。静态多态在编译阶段就确定了调用哪个函数或运算符。

开销：

  • 动态多态：在运行时需要额外的开销，因为需要在运行时查找虚函数表并调用正确的函数。
  • 静态多态：在编译时确定了调用的函数，没有额外的运行时开销。

调用时机：

  • 动态多态：函数调用发生在运行时，根据对象的实际类型来确定调用的函数。
  • 静态多态：函数调用发生在编译时，根据函数或运算符的参数类型来确定调用的函数。

实现方式：

  • 动态多态：使用虚函数和基类指针或引用来实现。
  • 静态多态：使用函数重载和运算符重载来实现。

virtua机制

虚函数（virtual function）机制是通过虚函数表（vtable）和虚函数指针（vpointer）来实现的。虚函数机制允许在运行时动态决定调用哪个函数，实现了动态多态性。

虚函数的实现原理如下：

  1. 虚函数表（vtable）：

    • 对于包含虚函数的类，编译器会在类的内部创建一个虚函数表（vtable）。虚函数表是一个指针数组，其中每个元素对应一个虚函数。
    • 虚函数表中的每个元素是一个函数指针，指向该虚函数在类中的实际定义。每个类只有一个虚函数表，其中保存了该类及其所有派生类中的虚函数地址。

  2. 虚函数指针（vpointer）：

    • 对于包含虚函数的类的每个对象，编译器会在对象的内存布局中添加一个虚函数指针（vpointer）。
    • 虚函数指针指向该对象所属的类的虚函数表。通过虚函数指针，程序可以在运行时找到正确的虚函数表，并调用适当的虚函数。

  3. 虚函数调用过程：

    • 当通过基类指针或引用调用虚函数时，实际上是通过该指针或引用所指向的对象的虚函数指针来查找虚函数表。
    • 通过虚函数指针可以确定所属的类的虚函数表，然后根据函数在虚函数表中的位置找到正确的虚函数，并进行调用。

纯虚函数和抽象类

它们用于定义接口，而不是实现具体的功能。这样可以使得派生类必须实现这些接口，从而实现了多态性。

抽象类

特点：

  • 抽象类是包含纯虚函数的类，不能实例化对象，只能被用作基类，用于定义接口。
  • 抽象类的存在是为了实现接口的规范，具体的功能实现由派生类完成。

纯虚函数

特点：

  • 纯虚函数是在基类中声明的虚函数，但没有提供实际的函数实现，只有函数的声明，而且在声明后面加上 = 0。
  • 纯虚函数没有函数体，因此不能直接在基类中调用该函数，它的具体实现必须在派生类中完成。
  • 一个类中如果包含了至少一个纯虚函数，这个类就被称为抽象类，不能实例化对象。
  • 派生类必须实现基类中的纯虚函数，否则派生类也会成为抽象类。

override关键字

在 C++11 及其之后的标准中，override 是一个关键字，用于显式地标记派生类中的虚函数对基类虚函数的重写（覆盖）。

通过在派生类中的虚函数声明后添加 override 关键字，可以明确告诉编译器该函数是用于重写基类中的虚函数，这样编译器在编译阶段就能检查是否存在匹配的虚函数，从而避免因函数名拼写错误或函数签名不匹配等问题而导致的错误。

虚析构和纯虚析构

在 C++ 中，虚析构函数和纯虚析构函数是用于处理多态性和抽象类的特殊类型的析构函数。

虚析构函数

  • 虚析构函数是在基类中声明为虚函数的析构函数。通过在基类的析构函数前面添加 virtual 关键字，可以将析构函数声明为虚函数。
  • 当使用基类指针指向派生类对象，并通过该指针删除对象时，如果基类的析构函数是虚函数，会在运行时调用正确的派生类的析构函数，确保正确释放派生类对象的资源。

纯虚析构函数

  • 纯虚析构函数是在基类中声明为纯虚函数的析构函数，它没有函数体，而且在声明后面添加 = 0。
  • 如果一个类中包含纯虚析构函数，这个类就是一个抽象类，不能实例化对象。
  • 纯虚析构函数的存在是为了实现接口的规范，确保派生类必须实现析构函数，防止内存泄漏。
