类和对象

在 C++ 中，类（Class）和对象（Object）是面向对象编程的核心概念。

类是一种用户自定义的数据类型，用于表示对象的抽象模板或蓝图。它定义了对象的属性（成员变量）和行为（成员函数）。类可以看作是一个数据结构，但不仅包含数据成员，还包含操作这些数据的方法。

对象是类的一个实例，通过类来创建一个对象。对象是类的具体化，它包含了类定义中定义的各个成员变量的实际值。可以将对象视为类的实体，代表现实世界中的具体事物。

类和对象的关系：

  • 类是模板：类是一种自定义的数据类型，它定义了对象的属性和行为。它仅仅是一个模板或者说蓝图，描述了对象应该具有的属性和行为。
  • 对象是实例：对象是类的实例化，它是类的具体表现，包含了类定义中定义的各个成员变量的实际值。通过类来创建对象。
  • 多个对象：可以根据一个类创建多个对象，每个对象是独立的，具有相同的属性和行为，但其实际值可以不同。

class和struct的区别

在C++中，class和struct是两种用于定义自定义数据类型的关键字，它们在语法上有一些区别，但在功能上几乎是相同的。

  1. 成员访问权限：

    • **class中默认的成员访问权限是private，而struct中默认的成员访问权限是public**。
    • 在class中定义的成员，默认情况下不能被类外部访问，而在struct中定义的成员，默认情况下可以被类外部访问。

  2. 继承：

    • 无论是class还是struct，都可以用于继承其他类或结构体。
    • 使用class时，继承的默认访问权限是private；使用struct时，继承的默认访问权限是public。

综上所述，class和struct在C++中的区别主要在于成员的默认访问权限。

类成员的访问权限

在C++中，类的访问权限用于控制类成员的可访问性。C++中有三种类的访问权限，分别是：public、protected和private。

public

  • 在类内部和外部，所有地方都可以访问公有成员。
  • 公有成员可以通过对象直接访问或通过类的对象或指针访问。
  • 公有成员通常用于定义类的接口，即类对外暴露的功能。

protected

  • 在类内部，所有地方都可以访问受保护成员；在类外部，只有派生类可以访问受保护成员。
  • 受保护成员不能直接通过对象访问，但可以通过派生类的对象或指针访问。
  • 受保护成员通常用于定义类的共享属性和行为，但对外部不可见。

private

  • 在类内部可以访问私有成员，但类的外部无法直接访问私有成员。
  • 私有成员只能通过类的成员函数来访问。
  • 私有成员通常用于定义类的内部实现细节，对外部完全隐藏。

构造函数和析构函数

在C++中，构造函数和析构函数是特殊的成员函数，用于在对象创建和销毁时执行一些特定的操作。构造函数用于初始化对象的成员变量，而析构函数用于释放对象在创建过程中分配的资源。

构造函数

  • 构造函数是一个特殊的成员函数，它的名称与类名相同，没有返回类型（甚至没有void），并在对象创建时自动调用。
  • 构造函数可以有多个重载版本，每个版本可以接受不同的参数列表，以便在创建对象时提供不同的初始化方式。
  • 如果没有定义构造函数，编译器将自动生成默认构造函数，它会执行默认的初始化操作。

  • 使用ClassName objectName = otherObject的方式拷贝初始化对象时，会调用拷贝构造函数。

  • 使用new运算符动态分配对象时，会调用合适的构造函数。

构造函数的调用规则

在C++中，构造函数在对象创建时自动调用，用于初始化对象的成员变量。构造函数的调用规则如下：

  1. 对象的创建时机：
    构造函数在以下情况下会被调用：

    • 在栈上创建对象：直接在函数内或作用域内定义对象时，构造函数会在对象创建时调用。
    • 在堆上创建对象：使用 new 运算符动态分配内存来创建对象时，构造函数会在对象创建时调用。

  2. 构造函数的调用顺序：
    如果一个类继承自其他类，则构造函数的调用顺序如下：

    • 首先，调用基类的构造函数，按照继承链的顺序从最基类开始向派生类逐级调用构造函数。
    • 然后，调用派生类自身的构造函数。

    析构函数的调用顺序与构造函数相反，先调用派生类析构函数，再调用基类析构函数。

匿名类

在C++中，匿名类指的是没有类名的局部类（Local Class），它是在函数内或块内定义的类，没有被命名，仅在定义的作用域内有效。匿名类用于在特定的局部作用域中封装一些功能，它的使用相对较少，但也有一些注意事项需要考虑：

  1. 作用域：

    • 匿名类的作用域被限制在它所定义的函数或块内部，超出这个作用域，它将不再可见。
    • 匿名类一般用于封装一些临时的或局部的功能，不需要在其他地方重用它。

  2. 访问权限：

    • 匿名类可以定义在函数的内部或块的内部，但不允许在命名空间中定义。
    • 由于匿名类没有类名，因此无法在其他地方通过类名来访问它。

  3. 无法创建对象：

    • 由于匿名类没有类名，无法直接创建匿名类的对象。
    • 通常情况下，匿名类的目的是为了利用类的封装性，定义一些私有成员和方法，并通过函数返回值、局部变量等方式来使用这些功能。

深拷贝和浅拷贝

  • 浅拷贝只复制对象中的成员变量的值，如果对象包含指针类型的成员变量，浅拷贝只复制指针的值，而不会复制指针指向的数据。
  • 浅拷贝会导致多个对象共享同一块内存，可能会导致潜在的问题，如资源释放不当或重复释放等。

  • 自定义拷贝构造函数时，要确保对资源进行正确的复制和释放，避免内存泄漏和野指针等问题。

  • 避免过度嵌套：
    尽量避免过度嵌套，避免出现过于复杂的类层次结构。

    静态成员在程序运行期间存在，直到程序结束时才会被销毁。在类的所有对象被销毁后，静态成员仍然存在，直到程序结束。
  • 静态成员的存储位置：
    静态成员变量在内存中只有一份拷贝，并存储在全局数据区或静态数据区。

在 C++ 中，类的成员函数和成员变量是分开存储的，它们在内存中占用不同的空间。

成员函数存储：

  • 类的成员函数（包括普通成员函数和静态成员函数）是存储在代码区（Code Segment）的，而不是类的实例中。当程序被加载到内存并执行时，代码区中存储了所有的函数定义，包括类的成员函数。
  • 每个成员函数只有一份拷贝，无论创建多少个类的对象，所有对象共享相同的成员函数代码。成员函数不会为每个对象实例分别存储，而是由所有对象共享。

成员变量存储：

  • 类的成员变量在对象实例中存储，它们被存储在栈区或者堆区中，取决于对象是在栈上还是通过 new 关键字动态创建在堆上。
  • 每个类的对象都会有自己的成员变量，每个对象实例中都有一份成员变量的拷贝。

const修饰成员函数

在 C++ 中，const 修饰成员函，它表示该成员函数在执行过程中不会修改对象的成员变量。被 const 修饰的成员函数被称为常量成员函数。使用 const 修饰成员函数的主要目的是为了确保在该函数内部不会对对象的状态进行修改。

常量成员函数的声明方式为在函数声明的末尾加上 const 关键字。常量成员函数不会改变调用它的对象，因此在常量成员函数内部，成员变量不能被修改（除非是 mutable 成员变量）。

  • 在常量成员函数中，不能修改除了 mutable 成员变量之外的成员变量。

  • 常量成员函数适用于对对象的读操作，而非常量成员函数适用于对对象的写操作。

  • 常量成员函数应该尽量返回成员变量或其他成员对象的常量引用，以避免不必要的复制。

在 C++ 中，mutable 是一个关键字，用于在常量成员函数中修改被声明为 mutable 的成员变量。使用 mutable 关键字可以绕过常量性检查，允许在常量成员函数中修改特定的成员变量，即使该函数是被常量对象调用的。
