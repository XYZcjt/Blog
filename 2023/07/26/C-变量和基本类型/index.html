cerr和clog

在C++中，cerr和clog都是用于输出错误消息和日志消息的流对象。它们都是标准C++库中的对象，提供了类似于cout的功能，但用于不同的输出目的。

  1. cerr（Standard Error）：
    cerr是标准C++库提供的预定义输出流，用于输出错误消息。与cout不同，cerr默认将输出发送到标准错误输出设备，通常是终端或控制台。因为cerr是不缓冲的流，所以当你使用cerr输出错误消息时，它会立即显示在终端上。

  1. clog（Standard Log）：
    clog是标准C++库提供的预定义输出流，用于输出日志消息。与cout和cerr不同，clog默认将输出发送到标准错误输出设备，通常是终端或控制台。与cerr类似，clog也是不缓冲的流，所以当你使用clog输出日志消息时，它会立即显示在终端上。

  • 使用cerr输出错误消息，它会立即显示在终端上，适合用于报告运行时错误和异常。
  • 使用clog输出日志消息，它也会立即显示在终端上，适合用于记录程序运行过程中的信息和状态。

字面值

在C++中，字面值（Literal）是指直接表示某种常量值的符号或字符串，而不是通过变量或表达式计算得到的值。C++支持多种类型的字面值，包括整数字面值、浮点数字面值、字符字面值、字符串字面值等。

以下是C++中常见的字面值类型：

  1. 整数字面值：例如 42、-123、0xFF 等。整数字面值可以是十进制、八进制、十六进制等表示方式。

  2. 浮点数字面值：例如 3.14、-0.001、2.0e-5 等。浮点数字面值可以是带小数点的十进制数，也可以使用科学计数法表示。

  3. 字符字面值：例如 'a'、'B'、'%' 等。字符字面值由单引号括起来，表示一个字符。

  4. 字符串字面值：例如 "Hello, World!"、"C++ Programming" 等。字符串字面值由双引号括起来，表示一个字符串。

  5. 布尔字面值：true 和 false，表示布尔类型的常量。

  6. 空指针字面值：nullptr，表示空指针。

  7. 用户自定义字面值：C++11引入了用户自定义字面值的功能，允许程序员定义自己的字面值常量。

嵌套作用域

在C++中，嵌套作用域指的是在一个代码块（例如函数、循环、条件语句等）内部定义的变量或对象只在该代码块内部有效，超出该代码块的范围就无法访问或使用。

嵌套作用域的主要特点包括：

  1. 变量的生命周期：在嵌套作用域内部定义的变量的生命周期仅限于该代码块内，当代码块执行结束后，这些变量会被销毁。

  2. 变量的可见性：在嵌套作用域内部定义的变量只能在该代码块内部访问，超出该代码块的范围无法访问这些变量。

  3. 命名冲突：在嵌套作用域中，可以使用相同的变量名或对象名，因为它们不会相互干扰，彼此独立。

NULL和nullptr

在C++中，NULL 和 nullptr 都是用来表示空指针的关键字，但它们有一些不同之处。

  1. NULL：
  • NULL 是C++的旧式表示空指针的方法，实际上是一个整数常量0的宏定义。
  • NULL 定义在 <cstddef> 或 <stddef.h> 头文件中，通常情况下，编译器会将 NULL 替换为整数0。
  • NULL 并不是真正的空指针类型，而是一个整数类型，因此在某些情况下可能会导致类型不匹配的问题。
  1. nullptr：
  • nullptr 是C++11引入的新的空指针表示方法，是一个关键字。
  • nullptr 是真正的空指针类型，可以用来初始化指针或与指针进行比较，避免了类型不匹配的问题。
  • nullptr 可以显式地用于初始化指针：int* ptr = nullptr;

new和delete

在C++中，new和delete是用于动态内存管理的操作符。

  1. new操作符：

    • 用于在堆上动态分配内存，创建对象或数组。
    • 语法：new 类型 或 new 类型[大小]
    • 返回一个指向分配内存的指针。
    • new操作符会调用构造函数来初始化对象。
    • 必须手动调用delete来释放分配的内存，否则会导致内存泄漏。

  2. delete操作符：

    • 用于释放使用new操作符分配的内存。
    • 语法：delete 指针 或 delete[] 指针
    • delete操作符会调用析构函数来销毁对象，并释放占用的内存。
