函数的默认参数

函数的占位参数

C++ 中的占位参数是指在函数声明中使用一个没有标识符名称的形式参数，通常用一个单一的下划线 _ 来表示。占位参数是一种特殊的形式参数，它在函数的实现中并不使用，只是为了满足函数的调用要求而存在。

占位参数在函数声明中起到以下作用：

  1. 占位参数用于表示函数的某个参数是必需的，但在函数的实现中并不使用该参数。
  2. 占位参数可以帮助我们实现函数的重载，因为函数的参数列表中如果只有类型不同而参数名相同的情况是不允许的，而使用占位参数可以避免这种问题。

  • 占位参数只在函数声明中使用，函数定义中不需要提供参数名，因为在函数实现中并不使用该参数。
  • 在一个函数的参数列表中，只能有一个占位参数，否则会导致编译错误。
  • 占位参数的目的是为了满足函数的重载需求，如果不需要实际使用该参数，可以使用占位参数。如果需要使用该参数，则应该使用正常的函数参数，并提供参数名。

总结来说，占位参数是一种特殊的形式参数，它在函数的实现中并不使用，仅用于函数的声明和重载。通过使用占位参数，我们可以实现更加灵活的函数重载和调用。

引用作为函数重载条件

在 C++ 中，引用可以作为函数重载的条件，就像其他参数类型一样。函数重载时，参数列表的类型和数量必须不同，引用也是一种参数类型，因此可以用引用来进行重载。

当函数重载涉及到引用作为参数时，需要注意以下几点：

  1. 常规引用和常量引用可以作为重载条件：普通引用（T&）和常量引用（const T&）是不同的参数类型，可以作为函数重载的条件。

  2. rvalue 引用不会与常规引用发生重载：rvalue 引用（右值引用，T&&）和普通引用是不同的类型，它们不会发生重载关系。

  3. const 修饰的引用不能与非 const 引用发生重载：const T& 和 T& 不能构成重载关系，因为它们在调用时可能会发生二义性。
