socket

在C语言中，Socket是一种用于网络通信的编程接口（API）。它允许程序员创建客户端和服务器应用程序，使它们能够在网络上进行数据传输和通信。

Socket编程允许两个或多个计算机之间通过网络进行通信。一个计算机可以充当服务器，提供某种服务或资源，而另一个计算机可以充当客户端，请求或使用这些服务或资源。通过Socket编程，可以实现客户端和服务器之间的双向数据传输。

大端字节序（Big Endian）

在大端字节序中，较高的有效字节（Most Significant Byte，MSB）存储在低地址，而较低的有效字节（Least Significant Byte，LSB）存储在高地址。这类似于从左到右阅读多字节数据，其中最重要的字节位于最左边。

小端字节序（Little Endian）

在小端字节序中，较低的有效字节（LSB）存储在低地址，而较高的有效字节（MSB）存储在高地址。这类似于从左到右阅读多字节数据，其中最重要的字节位于最右边。

不同的计算机体系结构使用不同的字节序。例如，x86架构的个人计算机通常采用小端字节序，而网络协议通常采用大端字节序。因此，在进行跨平台数据交换时，需要考虑字节序的差异，以确保数据在不同计算机上正确解释和处理。

inet_pton

在 C 语言中，可以使用 <arpa/inet.h> 头文件中的 inet_pton 和 inet_ntop 函数来进行字符串类型的 IP 地址和二进制网络地址之间的转换。这两个函数用于在网络编程中，将 IP 地址在字符串表示和二进制表示之间进行转换。

参数：

  • af: 地址族(IP 地址的家族包括 ipv4 和 ipv6)协议

    • AF_INET: IPv4 格式的 ip 地址

    • AF_INET6: IPv6 格式的 ip 地址

  • src: 传入参数，对应要转换的点分十进制的 IP地址(字符串)

  • dst: 传出参数，指向存储转换后的二进制网络地址的缓冲区的指针。

返回值：

  • 如果转换成功，返回1。
  • 如果转换失败，返回0或-1。

sockaddr的数据结构

在 C 语言中，sockaddr 是一个通用的套接字地址结构，用于表示各种类型的网络地址，包括 IPv4 地址和 IPv6 地址。

在 <sys/socket.h> 头文件中定义了 sockaddr 结构体及其变体结构体，如 sockaddr_in 和 sockaddr_in6，用于表示 IPv4 和 IPv6 地址。通常，在使用 bind、connect 等函数时，需要将具体的地址类型强制转换为 sockaddr 类型，并指定地址的大小，以便于函数处理。

以下是 sockaddr 结构体的定义：

sockaddr 结构体包含两个成员：

  1. sa_family：表示地址家族（Address Family），例如 AF_INET 表示 IPv4 地址家族，AF_INET6 表示 IPv6 地址家族。不同的地址家族将决定在 sa_data 中实际存储的地址信息的结构和大小。

  2. sa_data：用于存放地址的具体内容，长度为 14 个字节，通常用于存放 IP 地址或其他地址信息。

在实际使用中，通常会使用具体的变体结构体来表示不同类型的套接字地址，例如：

  • struct sockaddr_in：用于表示 IPv4 地址的结构体。
  • struct sockaddr_in6：用于表示 IPv6 地址的结构体。

这些变体结构体在 sa_family 字段的基础上提供了更多的字段来存放具体的地址信息，用于存储具体的 IPv4 或 IPv6 地址信息，而 sockaddr 结构体只是作为一个通用的容器，在需要时进行类型转换。

套接字函数

在 C 语言中，套接字函数（Socket Functions）用于进行网络通信，创建、配置、连接、发送和接收数据等操作。使用套接字通信函数需要包含头文件 <arpa/inet.h>，包含了头文件 <sys/socket.h> 。

以下是一些常用的套接字函数：

  1. socket()：创建一个套接字。

参数:

  • domain: 使用的地址族协议

    • AF_INET: 使用 IPv4 格式的 ip 地址
    • AF_INET6: 使用 IPv4 格式的 ip 地址

  • type:

    • SOCK_STREAM: 流式套接字(用于 TCP)
    • SOCK_DGRAM: 数据报套接字(用于 UDP)

  • protocol: 一般写 0 即可(默认协议，通常根据 domain 和 type 自动选择)

  • SOCK_STREAM: 流式传输默认使用的是 tcp

  • SOCK_DGRAM: 报式传输默认使用的 udp

返回值:

  • 成功：如果套接字创建成功，返回一个非负整数值，表示新创建的套接字描述符。
  • 失败: -1
  1. bind()：bind 函数用于将一个套接字与指定的地址（IP 地址和端口号）绑定在一起，使得套接字可以监听指定地址上的连接请求或者在指定地址上进行数据传输。

参数:

  • sockfd: 监听的文件描述符，通过 socket () 调用得到的返回值
  • addr: 一个指向 struct sockaddr 结构体的指针，表示要绑定的地址信息。由于 struct sockaddr 是一个通用的套接字地址结构体，通常需要将其转换为具体的地址结构体，如 struct sockaddr_in（IPv4 地址）或 struct sockaddr_in6（IPv6 地址）。
  • addrlen: 表示 addr 结构体的大小，可以使用 sizeof 运算符获取，例如sizeof (struct sockaddr)。

返回值：

  • 成功：返回 0
  • 失败：返回 - 1
  1. listen()：listen 函数用于将一个已绑定的套接字（通常是服务器套接字）设置为监听状态，以接受客户端的连接请求，通常在服务器端使用。

参数:

  • sockfd: 要设置监听状态的套接字描述符，即通过 socket 函数创建的套接字，并通过 bind 函数绑定到特定地址和端口。
  • backlog: 同时能处理的最大连接要求，最大值为 128

返回值：

  • 成功：返回 0
  • 失败：返回 -1
  1. accept()：accept 函数用于接受客户端的连接请求，并返回一个新的套接字描述符，该套接字描述符用于与客户端进行通信。accept 函数通常在服务器端使用，用于建立与客户端之间的连接。

参数:

  • sockfd: 监听状态的套接字描述符，即通过 socket 函数创建的套接字，并通过 bind 和 listen 函数准备好接受连接请求的套接字。
  • addr: 用于存储客户端的地址信息，可以为 NULL，表示不获取客户端的地址信息。如果不为 NULL，则需要传递一个指向 struct sockaddr 结构体的指针，accept 函数将填充客户端的地址信息。
  • addrlen: 表示 addr 结构体的大小，在调用 accept 函数之前需要设置为客户端地址结构体的实际大小，调用后会被修改为实际客户端地址结构体的大小。

返回值：

  • 如果接受连接成功，返回一个新的套接字描述符，用于与客户端进行通信。这个新的套接字描述符可以通过该套接字进行数据传输和通信。
  • 调用失败返回 -1

这个函数是一个阻塞函数，当没有新的客户端连接请求的时候，该函数阻塞；当检测到有新的客户端连接请求时，阻塞解除，新连接就建立了，得到的返回值也是一个文件描述符，基于这个文件描述符就可以和客户端通信了。

  1. connect()：connect 函数用于在客户端与服务器端建立连接。connect 函数通常在客户端程序中使用，用于连接到服务器，并与服务器进行通信。

参数:

  • sockfd: 客户端套接字描述符，即通过 socket 函数创建的套接字。
  • addr: 指向服务器地址信息的指针，通常是一个 struct sockaddr 结构体的指针。该结构体包含了服务器的 IP 地址和端口号等信息。
  • addrlen: 表示 addr 结构体的大小，通常可以使用 sizeof 运算符获取，例如sizeof (struct sockaddr)

返回值：

  • 成功：返回 0
  • 失败：返回 - 1
  1. 接收数据

参数：

  • sockfd: 要读取数据的文件描述符，通常是套接字描述符。
  • buf: 用于存储读取数据的缓冲区。
  • size: 要读取的最大字节数。
  • flags: 特殊的属性，一般不使用，指定为 0。

返回值：

  • 大于 0：实际接收的字节数
  • 等于 0：对方断开了连接
  • -1：接收数据失败了

区别：

  • read 函数是标准的 UNIX 系统调用，在文件描述符上通用，包括套接字。
  • recv 函数是专门用于套接字的系统调用，比 read 函数稍微复杂，因为它支持更多的功能，如通过 flags 参数控制读取行为。（建议使用）
  • recv 函数除了用于普通的套接字读取数据外，还可以在 TCP 套接字上使用一些特殊的标志参数，例如 MSG_WAITALL 用于保证接收完整的数据块。

如果连接没有断开，接收端接收不到数据，接收数据的函数会阻塞等待数据到达，数据到达后函数解除阻塞，开始接收数据。当发送端断开连接，接收端无法接收到任何数据，但是这时候就不会阻塞了，函数直接返回0。

  1. 发送数据

参数:

  • fd: 要写入数据的文件描述符，通常是套接字描述符。
  • buf: 包含要发送的数据的缓冲区。
  • len: 要发送的字节数。
  • flags: 特殊的属性，一般不使用，指定为 0。

返回值：

  • 大于 0：实际发送的字节数，和参数 len 是相等的
  • -1：发送数据失败了

区别：

  • write 函数是标准的 UNIX 系统调用，在文件描述符上通用，包括套接字。
  • send 函数是专门用于套接字的系统调用，比 write 函数稍微复杂，因为它支持更多的功能，如通过 flags 参数控制发送行为。（建议使用）
  • send 函数除了用于普通的套接字发送数据外，还可以在 TCP 套接字上使用一些特殊的标志参数，例如 MSG_NOSIGNAL 用于避免向已关闭的连接发送数据时触发 SIGPIPE 信号。

服务器端通讯流程

1.创建用于监听的套接字，这个套接字是一个文件描述符

2.将得到的监听的文件描述符和本地的 IP 端口进行绑定

5.通信，读写操作默认都是阻塞的

6.断开连接，关闭套接字

INADDR_ANY常量

在 C 语言中，INADDR_ANY 是一个特殊的常量，用于表示一个服务器套接字可以绑定到任意可用的本地 IP 地址。它是在 <netinet/in.h> 头文件中定义的，通常用于服务器程序中，用于指定服务器监听的 IP 地址。

INADDR_ANY 的值是 0，它对应的是 IPv4 地址 0.0.0.0。当服务器套接字使用 INADDR_ANY 绑定时，表示服务器将接受任意到达本地的连接请求，不论是来自本地主机上的地址，还是来自其他主机的地址。

在使用 bind 函数绑定服务器套接字时，可以将服务器的 IP 地址设置为 INADDR_ANY，以便服务器可以接受来自本地和远程主机的连接请求。

多线程并发

上面实现的通信程序，只能供一个客户端使用，如果要涉及到多个客户端同时访问，就要使用多线程解决服务器并发问题。

多线程中的线程注意有两大类：主线程和子线程，他们在服务器端分别处理监听和通信流程，可以按照以下思路进行设计：

主线程：

  • 负责监听，处理客户端的连接请求，也就是在父进程中循环调用accept()函数
  • 创建子线程：建立一个新的连接，就创建一个新的子进程，让这个子进程和对应的客户端通信
  • 回收子线程资源：由于回收需要调用阻塞函数，这样就会影响accept()，直接使用pthread_detach()函数做线程分离即可。

子线程：负责通信，基于主线程建立新连接之后得到的文件描述符，和对应的客户端完成数据的接收和发送。

  • 发送数据：send() / write()
  • 接收数据：recv() / read()
