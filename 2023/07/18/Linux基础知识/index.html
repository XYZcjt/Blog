1.10重定向命令

文件重定向是一种在命令行中将输入和输出重定向到文件的技术，用于将命令的输入和输出从终端（标准输入和标准输出）改为文件。在Linux和类Unix系统中，文件重定向使用特定的符号来实现。

在命令行中，有三个主要的文件描述符与文件重定向相关联：

  1. 标准输入（stdin）：文件描述符为0，用于读取程序的输入。默认情况下，它连接到终端的键盘输入。

  2. 标准输出（stdout）：文件描述符为1，用于向屏幕输出程序的结果。默认情况下，它连接到终端的显示器。

  3. 标准错误（stderr）：文件描述符为2，用于向屏幕输出程序的错误消息。默认情况下，它连接到终端的显示器。

常用的文件重定向符号如下：

  • >：将标准输出重定向到指定文件，如果文件不存在则创建，如果文件已存在则覆盖原内容。
  • >>：将标准输出重定向追加到指定文件，如果文件不存在则创建，如果文件已存在则在末尾添加内容。
  • <：将指定文件作为标准输入，用于提供程序的输入数据。
  • 2>：将标准错误重定向到指定文件，用于捕获程序的错误消息。
  • 2>>：将标准错误重定向追加到指定文件，用于追加捕获程序的错误消息。

2.文件

2.1文件类型

在Linux系统中，每个文件都有一个文件类型（File Type）标识，用于指示文件的类型。文件类型通常表示文件是普通文件、目录、符号链接、设备文件等。要查看文件类型，可以使用ls命令的-l选项，或者使用file命令。

  1. 普通文件（Regular file）： 这是最常见的文件类型，用于存储文本、数据或二进制信息。普通文件可以是文本文件、配置文件、可执行文件等。普通文件以-字符开头。

  2. 目录（Directory）： 用于存储其他文件和子目录。目录以d字符开头。

  3. 符号链接（Symbolic Link）： 也称为软链接，它是一个指向另一个文件或目录的快捷方式。符号链接以l字符开头。

  4. 块设备文件（Block Device）： 用于访问块设备，如硬盘驱动器。块设备文件以b字符开头。

  5. 字符设备文件（Character Device）： 用于访问字符设备，如串口、键盘等。字符设备文件以c字符开头。

  6. 套接字（Socket）： 用于进程间的网络通信。套接字以s字符开头。

  7. 命名管道（Named Pipe）： 也称为FIFO（First In First Out），用于进程间通信。命名管道以p字符开头。

  8. 特殊文件（Special file）： 用于其他特殊目的，例如挂载点（Mount Point）。在ls -l命令输出中，特殊文件以-字符开头，但在第一个字符之后还有其他标识。

2.2文件权限

在Linux系统中，每个文件和目录都有权限控制，用于确定谁可以读取、写入或执行它们。文件权限由三个组成部分组成：所有者权限、群组权限和其他用户权限。

每个权限部分可以分为三个标志位：读取权限（r）、写入权限（w）和执行权限（x）。以下是文件权限的含义：

  • 读取权限（r）：表示能够查看文件的内容，对于目录来说，表示能够列出目录中的文件和子目录。
  • 写入权限（w）：表示能够修改文件的内容，对于目录来说，表示能够在目录中创建、删除和重命名文件和子目录。
  • 执行权限（x）：表示能够执行文件，对于目录来说，表示能够进入该目录。

文件权限可以用符号表示法或数字表示法来表示。符号表示法使用字母（r、w、x）和符号（-）来表示权限。每个权限组使用三个字母表示，分别表示所有者权限、群组权限和其他用户权限。例如，-rw-r--r-- 表示文件所有者具有读写权限，群组和其他用户具有只读权限。

3.Linux目录结构

Linux的目录结构是一种标准化的层次化文件系统结构，它采用了类似于树状的布局，使得文件和目录可以按照一定规则组织和管理。以下是Linux的主要目录结构：

  1. /（根目录）： 整个文件系统的起始点，所有文件和目录都位于根目录下。

  2. /bin： 存放系统命令和二进制可执行文件。一般用户和管理员都可以使用这些命令。

  3. /boot： 存放启动加载器和内核镜像等引导文件。

  4. /dev： 存放设备文件，用于表示和访问系统硬件设备。

  5. /etc： 存放系统配置文件，如网络配置、用户配置等。

  6. /home： 用户的主目录，每个用户都有一个独立的子目录。

  7. /lib和/lib64： 存放共享库文件，这些库文件被程序用于运行时链接。

  8. /media： 临时挂载点目录，用于挂载可移动设备，如CD-ROM、USB等。

  9. /mnt： 临时挂载点目录，用于挂载其他文件系统。

  10. /opt： 用于存放可选的第三方软件。

  11. /proc： 虚拟文件系统目录，提供有关系统和进程的信息。

  12. /root： 超级用户（root）的主目录。

  13. /run： 存放系统启动后运行时需要的临时文件。

  14. /sbin： 存放系统管理员命令，一般只有超级用户（root）可以使用。

  15. /srv： 存放服务相关的数据。

  16. /sys： 虚拟文件系统目录，提供内核和设备信息。

  17. /tmp： 存放临时文件，系统重启后该目录内容会被清空。

  18. /usr： 存放用户应用程序和文件，类似于Program Files目录。

  19. /var： 存放经常变化的文件，如日志文件、缓存文件等。

5.1用户类型

在Linux系统中，每个文件和目录都有一个所有者（Owner）、一个所属组用户（Group）和其他人（Others）这三种权限级别。这些权限级别决定了谁可以访问文件或目录以及拥有何种访问权限。每个文件或目录都有各自的权限设置。

1. 文件所有者（Owner）：
文件所有者是指创建或拥有该文件或目录的用户。文件所有者可以对文件或目录设置不同的权限，包括读取（r）、写入（w）和执行（x）权限。只有文件所有者可以更改文件的所有权或权限设置。

2. 文件所属组用户（Group）：
文件所属组用户是指与文件或目录相关联的用户组。所有组成员都拥有该组的权限。与文件所有者类似，文件所属组用户也可以设置文件或目录的权限。

3. 其他人（Others）：
其他人是指不是文件所有者，也不属于文件所属组用户的用户。这包括系统中的所有其他用户。其他人权限是指所有不属于文件所有者和文件所属组用户的用户的权限。

在Linux中，可以通过ls -l命令查看文件或目录的权限信息。输出的权限信息如下所示：

  • 1：表示文件的链接数，意思是在内存空间新创建了一块数据，然后会有一个文件名字，这个文件名字就是这个数据的索引，可以通过索引找到这个数据。如果有多个索引，不同索引都是指向的这个空间，相当于是共享了这一块空间，通过ls -l命令输出是，链接数就大于1。

5.3退出超级用户

在Linux中，退出超级用户（root）模式并返回到普通用户模式，可以通过以下方式实现：

  1. 使用exit命令： 最常见的方法是在超级用户模式下直接使用exit命令。这将立即关闭超级用户终端会话并返回到之前的普通用户会话。

  2. 使用logout命令： 与exit类似，也可以使用logout命令来退出超级用户模式并返回到普通用户模式。

  3. 使用Ctrl+D组合键： 在超级用户终端中，可以使用Ctrl+D组合键来退出超级用户模式，返回到普通用户模式。

6.PATH环境变量

在Linux中，$PATH是一个环境变量，用于指定系统搜索可执行程序的路径。当用户在终端中输入一个命令时，Linux会根据$PATH中列出的路径来查找并执行对应的可执行程序。

环境变量是一种存储在操作系统中的键值对，用于存储各种系统配置和运行时参数。环境变量在整个系统中都可用，用户可以通过设置、修改或查看环境变量来改变系统的行为。

$PATH环境变量由一系列以冒号（: ）分隔的目录路径组成。当用户在终端中输入一个命令时，Linux会按照$PATH中列出的顺序在这些目录中查找对应的可执行文件。一旦找到匹配的可执行文件，就会执行该程序。

7.一些常用命令行快捷键

以下是一些常用的Linux命令行快捷键：

  1. Tab键： 输入部分命令或文件路径后，按下Tab键可以自动补全命令或文件名，如果有多个匹配项，可以按两次Tab键显示所有匹配项。

  5. Ctrl + E： 将光标移动到行尾。

  8. Ctrl + Y： 粘贴之前使用Ctrl + U，Ctrl + K删除的文本。
  9. Ctrl + L： 清屏，相当于执行clear命令。

8.链接创建

在Linux系统中，有两种类型的链接：硬链接（Hard Link）和软链接（Symbolic Link，也称为符号链接）。它们都是用于将一个文件链接到另一个位置的机制，但它们之间有一些重要的区别。

硬链接（Hard Link）：

  1. 硬链接是指多个文件名链接到同一个索引节点（inode），它们共享相同的数据和元数据。
  2. 硬链接只能链接到同一个文件系统中的文件，不能跨越文件系统进行链接。
  3. 删除一个硬链接并不会影响其他硬链接，只有当所有硬链接都被删除时，文件的存储空间才会真正释放。
  4. 硬链接不能链接目录，只能链接普通文件。
  5. 硬链接没有权限问题，对于文件所有者、所属组用户和其他用户而言，所有的硬链接都具有相同的权限和访问控制。

创建硬链接的方法是使用ln命令，不带任何选项时为硬链接：

软链接（Symbolic Link）：

  1. 软链接是一个特殊的文件，它包含了指向目标文件的路径。软链接类似于Windows系统中的快捷方式。
  2. 软链接可以链接到同一文件系统中的文件，也可以跨越文件系统进行链接。
  3. 删除软链接不会影响原始文件，但如果删除了原始文件，则软链接将变为“坏链接”（Broken Link），无法访问目标文件。
  4. 软链接可以链接到目录，也可以链接到普通文件。
  5. 软链接具有自己的权限，对于文件所有者、所属组用户和其他用户而言，它们可能具有不同的权限和访问控制。

总结：

  • 硬链接链接到同一文件系统中的文件，不支持链接目录，不能跨越文件系统。
  • 软链接支持链接目录，可以跨越文件系统，但删除原始文件会导致软链接失效。
  • 硬链接的文件计数（硬连接计数）会增加，直到所有硬链接被删除才释放存储空间。
  • 软链接是独立的文件，不占用实际数据块，删除软链接不会影响目标文件。
